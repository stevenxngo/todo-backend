TODO API Server Documentation

API Endpoints

GET /todos

Retrieve a list of all TODO items.

GET /todos

This endpoint returns a JSON array of all TODO items stored in the database. Each item includes its ID, title, and completion status.

Responses:

200 OK
[
  {
    "id": 1,
    "title": "Example Task",
    "completed": false
  }
]
500 Internal Server Error
{
  "error": "An error occurred"
}

POST /todos

Create a new TODO item.

POST /todos

Request Body:

{
  "title": "New Task"
}

The title is required to create a new TODO item. The response will include the ID of the newly created item and its initial completion status (false by default).

Responses:

201 Created
{
  "id": 1,
  "title": "New Task",
  "completed": false
}
400 Bad Request
{
  "error": "Title is required"
}

GET /todos/<id>

Retrieve a TODO item by ID.

GET /todos/1

This endpoint returns a JSON object representing the TODO item with the specified ID. If the item is not found, a 404 error is returned.

Responses:

200 OK
{
  "id": 1,
  "title": "Example Task",
  "completed": false
}
404 Not Found
{
  "error": "ToDo item not found"
}

PUT /todos/<id>

Update a TODO item by ID.

PUT /todos/1

Request Body:

{
  "title": "Updated Task",
  "completed": true
}

Update the title and completion status of an existing TODO item. If not provided, the title or completed fields will remain unchanged. The response includes the updated item details.

Responses:

200 OK
{
  "id": 1,
  "title": "Updated Task",
  "completed": true
}
404 Not Found
{
  "error": "ToDo item not found"
}

DELETE /todos/<id>

Delete a TODO item by ID.

DELETE /todos/1

A successful request will return a confirmation message. If the item is not found, a 404 error is returned.

Responses:

200 OK
{
  "message": "ToDo item deleted"
}
404 Not Found
{
  "error": "ToDo item not found"
}

For more details or additional functionalities, please refer to the API documentation or contact the administrator.
